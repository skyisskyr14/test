# 面向中小企业的员工绩效考核管理系统 - 运行说明

## 目录结构
- `spring/`：Spring Boot + MyBatis 后端工程
- `web/`：Vue3 + Element Plus 前端工程
- `data/`：数据库 SQL 初始化脚本
- `output/`：一键运行脚本与 Docker Compose 编排

## 当前已完成功能（优先功能健全）
1. 认证登录 + 会话 Token
2. 考核方案管理（新建、发布）
3. 方案指标分配（绑定指标及权重）
4. 方案参与人分配（自动生成任务）
5. 指标库管理（新增、删除、列表）
6. 员工任务中心（查看任务）
7. 员工自评打分（按指标提交得分）
8. 自动核算结果（生成等级 A/B/C/D）
9. 管理层结果审批（通过/驳回）
10. 绩效申诉（提交、处理）
11. 报表页（部门对比、趋势）
12. 报表图表（ECharts 柱状/折线图）
13. 角色分离导航与访问控制（员工/HR/管理层）
14. 项目进度看板（Roadmap）
15. 系统操作日志看板（HR/管理层）
16. 方案/指标/报表精细化交互（筛选、校验、导出）

## 方式一（推荐）：Docker 一键启动
```bash
cd output
./run.sh
```
前端：`http://<你的服务器IP>:5173`  后端：`http://<你的服务器IP>:8080/api`

停止：
```bash
cd output
./stop.sh
```

## 方式二：本地启动
1. 导入数据库：执行 `data/init.sql`
2. 启动后端：
```bash
cd spring
mvn spring-boot:run
```
3. 启动前端：
```bash
cd web
npm install
npm run dev
```

## 常见问题排查
1. **登录 500 / 后端容器退出**
   - 执行 `docker compose logs -f backend` 查看后端日志。
   - 若看到 `Could not resolve dependencies for com.mysql:mysql-connector-j` 或类似依赖错误，请先拉取最新代码后重启：
```bash
git pull
cd output
./stop.sh
docker compose up -d mysql
./run.sh
```

2. **前端能打开但接口失败**
   - 确认后端已启动并监听 8080：`docker ps | grep perf-backend`
   - 服务器防火墙放通 8080/5173：`ufw allow 8080 && ufw allow 5173`

3. **MySQL 初始化慢导致首次启动失败**
   - 首次启动时建议先只起 MySQL，等待 10~20 秒后再启动后端。

可用于部署后快速验证后端与数据库是否连通。


## 截图工具（Playwright）
已在前端工程中安装截图能力，可用于自动化验收截图。

安装浏览器（首次）：
```bash
cd web
npx playwright install chromium
npx playwright install-deps chromium
```

执行截图：
```bash
cd web
npm run screenshot -- http://127.0.0.1:5173 ../output/latest-screenshot.png
```


登录后页面截图（免手工登录）示例：
```bash
cd web
SEED_TOKEN=dummy SEED_USER='{"id":1,"username":"hr_admin","realName":"HR管理员","roleId":2}' \
  npm run screenshot -- http://127.0.0.1:5173/dashboard/roadmap ../output/roadmap-screenshot.png
```
